01

FIRST LAUNCH & SETUP

Google Sheets Sync

All user accounts, orders, and custom blueprints are stored in a Google Sheet you control. On first launch the URL field is blank — you must connect the kiosk to your own deployed Apps Script sheet before any data will load.

  1. Open your Google Sheet, go to Extensions → Apps Script, paste the SC_Kiosk_AppScript.js contents, and save.
  2. Click Deploy → New Deployment → Web App. Set Execute as: Me and Who has access: Anyone. Copy the Web App URL.
  3. In the kiosk, click SYNC in the top-right bar, paste the URL (must start with https://script.google.com/macros/s/ and end with /exec), then click Save & Connect.
  4. A ping test confirms the connection — green means live, red means the URL is unreachable or the Apps Script is not yet deployed correctly.
  5. Important: whenever you edit the Apps Script code, you must create a new deployment version (Deploy → Manage Deployments → pencil icon → New version). Editing the code alone does not update the live endpoint.

User Accounts

Users are stored in the SC_Users sheet. Each user has a Rank, RSI Handle, and optionally a Password. Select your name from the user dropdown in the Inventory tab, or use the LOGIN button in the top bar (see below) to activate your account from any tab.

ADMIN ACCOUNT

The Gnome Admin account (👑 crown) is the super-user. It can edit orders, manage users, and access all features.

Add userInventory → + ADD USER — enter rank, RSI handle, optional password
Delete userInventory → user dropdown → DELETE USER
Change passwordInventory → Change Password
Admin modeClick the GNOME ADMIN button in the top-right and enter the admin password

LOGIN Button

The LOGIN button sits in the top-right bar, to the left of the GNOME ADMIN button. It lets you log in or switch users from any tab — you no longer need to navigate to Inventory first.

  1. Click LOGIN in the top-right bar.
  2. A popup lists all user accounts. Users with a password show a 🔒 icon. The currently active user is highlighted with a LOGGED IN tag.
  3. Click your name. If your account has a password, the password prompt appears automatically. If not, you are logged in immediately.
  4. Once logged in, the button label changes from LOGIN to your first name in teal so you can always see who is active.
  5. To log out, open the LOGIN button again and click LOGOUT at the bottom of the popup.
NOTE

Logging in via the LOGIN button is identical to selecting a user in the Inventory tab — your blueprints, ore inventory, and permissions are all activated the same way.

02

FABRICATE TAB

The Fabricate tab is the main blueprint browser. 1,554 blueprints from patch 4.8.0 are embedded directly in the kiosk — no internet connection required to browse them.

Blueprint Browser

Blueprints are grouped by category → subcategory, both sorted alphabetically. Use the category icon bar at the top to jump to a section, or use the Search box to filter by name. The AFFORDABLE filter shows only blueprints you have enough ores to craft. The OWNED filter shows blueprints marked as owned by the selected user.

Category barClick an icon to jump to that category. Hover for subcategory dropdown.
SearchFilters blueprint names live as you type.
AFFORDABLEHides blueprints where your ore inventory is insufficient for any slot.
OWNEDHides blueprints not marked as owned by the current user.
Quantity− / + or Max — scales all material requirements.
BLUEPRINTS tabShows the blueprint grid.
MATERIALS tabShows ore cards for all materials needed across your current selection.

Materials Panel (right side)

Clicking a blueprint opens the detail panel on the right. The top half shows the item image with its name, category path, tier, and fabrication time. The bottom half shows the Statistics panel with real base values from the game files.

Ore Slots

Each blueprint has 1–3 ore slots. Each slot shows the part name (e.g. FRAME, EMITTER, APERTURE IRIS), the required ore, quantity, and unit. The slot header turns orange with "Not enough material" if your inventory is short.

  1. Click anywhere on the ore row to open the material selection modal. It shows your available ores of that type sorted by quality.
  2. Click an ore to assign it to the slot. The slot updates with the ore's name, average quality, and available quantity.
  3. Click the ✕ in the slot header to remove an assigned ore.
  4. Use Best material / Worst material to autofill all empty slots at once.
  5. Use Clear All to remove all assigned ores.

Quality Sliders

Below each ore slot is a quality slider (0–1000). At 500 (default) the item is crafted at standard stats. Moving the slider simulates using ore of that quality and updates the Statistics panel in real time.

Drag sliderClick and hold, then drag the thumb left or right — quality moves from 0 (worst) to 1000 (perfect). Stats update live. The hit area is intentionally tall to make grabbing easy.
Click the numberType an exact quality value and press Enter or click away.
Ore assignedSlider locks to the ore's actual quality (🔒). Remove the ore to adjust manually.
Stat effect previewBelow each slider a line shows which stats that slot affects and by how much at the current quality, e.g. Integrity +8.3%.
QUALITY DATA SOURCE

Quality modifiers are sourced directly from the game's crafting.zip data files (patch 4.8.0). All 38 ores have exact quantization curves and 1,465 blueprints have per-slot stat modifier ranges. The formula is:

mult = modLo + (modHi − modLo) × effectiveQ / 1000

where effectiveQ is the raw quality run through the ore's quantization table.

Statistics Panel

Stats are pulled from the game's scitem.zip item definition files. 1,496 blueprints have real numeric base values. The panel shows three columns:

Attribute Base Stats Enhanced Stats Delta
Fire Rate 180.00 RPM 196.16 RPM +16.16 RPM ▲
Impact Force +0.00 % +5.61 % +5.61 % ▲
Recoil Kick +0.00 % −31.68 % −31.68 % ▼

Base Stats = standard values at quality 500. Enhanced Stats = values at your current slider positions. Delta = the change, with green ▲ for improvements and red ▼ for reductions. Note that Recoil Kick and similar stats are negated — a lower value is better, so a reduction still shows green.

ORE QUALITY MODIFIERS (v8)

In v8, quality sliders live exclusively in the material slots panel (centre of the Fabricate screen). The Statistics panel no longer contains its own sliders — the stat preview updates automatically as you drag the sliders in the material slots. This keeps all quality controls in one place and eliminates the dual-slider confusion from earlier versions.

PLACING ORDERS

The Orders tab lets members request fabrication jobs. Orders are saved to the Google Sheet and picked up by artificers in the Fulfillment tab.

  1. Navigate to the Orders screen (clipboard icon in the left sidebar). Browse and click blueprints in the left panel to select them (teal highlight + checkmark). Clicking again deselects. You can also click + ADD TO ORDER directly from the Fabricate detail panel to add the currently-open blueprint without switching screens.
  2. Click + ADD TO ORDER in the Orders screen top bar (shows a count badge of how many are selected). The right panel updates immediately with ORDER ITEMS and TOTAL MATERIALS columns.
  3. Enter your Callsign and RSI Handle in the fields at the top of the right panel. Both are required — the PLACE ORDER button stays disabled until both are filled.
  4. Optionally add a note for the artificer — see Adding a Note below.
  5. Click PLACE ORDER. The order is saved to the Google Sheet with New Order status and appears in the Fulfillment screen immediately.

Adding a Note to Your Order

Before placing an order you can attach a free-text note — useful for special requests, delivery instructions, or priority flags that the artificer needs to know.

  1. In the Orders screen, click the ADD NOTE button in the top bar (to the left of PLACE ORDER).
  2. A note input bar expands below the callsign/RSI fields. Type your message — for example "Please prioritise — ship combat event this weekend".
  3. Click PLACE ORDER as normal. The note is saved with the order and visible to artificers in the Fulfillment tab.
TIP

The note field clears automatically after the order is submitted. If you close the Orders tab without placing the order, the note is discarded.

Gnome Admin Master Blueprint List

The Orders tab blueprint filter defaults to the ⚙ Gnome Admin Master List — a live union of every blueprint owned by any user with the artificer permission. This means you only see blueprints that can actually be crafted by your org's artificers, rather than the full database of 1,554 items.

Default filter⚙ Gnome Admin Master List — all artificer-owned blueprints combined
Change filterUse the user dropdown in the Orders tab to switch to a specific artificer's list or All Blueprints
How it updatesRebuilt automatically every time users sync from Google Sheets — no manual steps needed
Who controls itArtificers manage their own blueprint lists in the Inventory tab; the master list reflects their combined ownership
NOTE FOR ARTIFICERS

Keep your blueprint list in Inventory up to date. Any blueprint you toggle as owned is automatically included in the Gnome Admin Master List that all users see when placing orders.

Order Notes

Each order card has an ORDER NOTES button inline with the ORDER ITEMS & MATERIALS heading. This is where artificers can record progress updates, material substitutions, or any information relevant to that specific order.

  1. Expand an order card in the Fulfillment tab.
  2. Click ORDER NOTES next to the ORDER ITEMS & MATERIALS heading.
  3. A popup appears. Artificers and admins see an editable text area — type or update the note and click SAVE.
  4. Other users see the note as read-only. A message indicates that only Artificers can edit notes.
  5. Saved notes are written back to the Google Sheet immediately and persist across sessions.
PERMISSIONS

Editing order notes (and order status, assignee, delete) requires the artificer or admin permission. See the Roles section for details. All users can read notes.

Order Priority

Artificers can flag any order with a priority level to visually sort workload at a glance. The priority changes the border colour of the entire order card so high-urgency jobs stand out even when all cards are collapsed.

  1. On any order card header, click the P button — it sits immediately to the right of the ⚙ cog. The P button is only visible to users with artificer or admin permissions.
  2. A small dropdown opens just below the button with four options.
  3. Click the desired priority. The card border and P button colour update instantly — no page refresh needed.
  4. The priority is saved to the Google Sheet immediately.
HighRed border — urgent, needs immediate attention
MediumLight blue border — standard priority
LowGreen border — fulfil when available
NoneDefault border — no priority set
TIP

The P button itself takes on the current priority colour, so you can see an order's priority at a glance without opening the dropdown. A P button in the default muted colour means no priority is set.

The status filter and assignee filter (custom checkbox dropdowns) let you narrow the view to only what's relevant to you.

4b

ORE MANIFEST

The Ore Manifest is the org-wide ore ledger — accessible via the table icon in the left sidebar. This is the master stock list the kiosk uses for the AFFORDABLE filter and the Autofill buttons in the Fabricate screen. It is separate from the per-user ore entries in the Inventory tab.

Add oreClick + Add Ore or the + Add new ore entry row at the bottom of the table. A blank row appears inline — set the name, type, quality (0–1000), quantity (cSCU or units), and optional notes, then click SAVE.
Edit oreClick the EDIT button on any row to edit it inline. Press Enter or click SAVE to confirm. Click ✕ to discard.
Delete oresCheck the checkbox on any rows (or the header checkbox to select all), then click the ✕ Delete button. Confirm in the dialog.
Search & filterUse the search bar to filter by name or type. The All Types dropdown filters by ore category (Gas, Metal, Mineral, Natural, Nonmetal, Processed, Waste).
Hand-mined oresOres like Dolivine, Hadanite, and Aphorite are measured in units / rocks rather than cSCU. The kiosk detects these automatically and adjusts requirement calculations.
RELATIONSHIP TO INVENTORY TAB

The Ore Manifest is the org's shared stock pool. The Inventory tab tracks each individual user's personal ore contributions with per-entry quality values. When an ore is assigned to a fabrication slot, its quality comes from the Inventory entry — the Manifest quantity is used only for affordability checks and autofill matching.

The blueprint list in Inventory shows all 1,554 blueprints. Clicking a blueprint toggles your ownership of it. Owned blueprints appear with a teal highlight and are included when the OWNED filter is active in the Fabricate tab. Filter by category or use the search box to find specific blueprints. Viewing Another User's Inventory

The VIEW dropdown (to the right of the USER dropdown in the Inventory tab topbar) lets anyone browse any other user's blueprints and ore stocks — no login or password required, even if the account is password-protected.

  1. In the Inventory tab, find the VIEW: dropdown in the top bar, to the right of the USER dropdown.
  2. Select any user from the list. Their blueprints and ores load immediately.
  3. The top-right info bar changes to 👁 VIEWING: Rank Name (read-only) so it's always clear you are not in your own account.
  4. To return to your own inventory, set the VIEW dropdown back to — Select User —.
BlueprintsOwned blueprints show the teal ✓ checkmark as normal, but clicking a row does nothing — toggling is disabled in view mode.
OresAlways shows only ores that user has entered (equivalent to My Ores mode). The ore type filter and search still work.
Edit controlsThe edit cog on ore rows is hidden. The + ADD prompt is replaced with a dash. No data can be changed.
Your sessionYour own login is unaffected — you can be logged in as one user and viewing another's inventory at the same time.
USE CASE

Useful for artificers checking what ores a member has available before accepting an order, or for members checking whether a fellow artificer owns a specific blueprint before requesting it.

BLUEPRINT EDITOR

The Blueprint Editor lets admins and authorised users create custom blueprints — items not in the wiki database or org-specific recipes. Custom blueprints are stored in the SC_CustomBlueprints Google Sheet and appear alongside the standard blueprints in the Fabricate and Orders tabs.

  1. Log in as a user with editor access (admin role, admin permission, or Gnome Admin). The Blueprint Editor screen shows a lock state when no eligible user is logged in.
  2. Click + ADD BLUEPRINT in the Blueprint Editor screen toolbar.
  3. Fill in the name, category, subcategory, grade, and craft time.
  4. Add material slots — each slot needs a part name, ore type, quantity, and hand-mined flag.
  5. Click SAVE. The blueprint immediately appears in the Fabricate tab.
  6. Edit or delete existing custom blueprints from the same screen.
